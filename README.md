# Atividade — Réplica de páginas SAP

## Objetivo

Reproduzir, com **HTML** e **CSS**, duas páginas do **SAP**, buscando o maior rigor possível em relação aos modelos propostos, com foco em organização estrutural, alinhamento, espaçamento, dimensões, hierarquia visual e padronização de estilos.

## Capacidades avaliadas

- CT7: Reconhecer especificações técnicas e paradigmas de linguagem de programação na construção de interfaces com HTML e CSS.
- CS3: Organizar o projeto, os arquivos e o processo de desenvolvimento.

## Entregas

- [ ] Dois arquivos `.html` correspondentes às duas páginas solicitadas
- [ ] Dois arquivos `style.css` correspondentes às duas páginas solicitadas
- [ ] Reprodução visual das duas páginas propostas
- [ ] Estrutura organizada e padronizada entre os arquivos
- [ ] Código funcionando no navegador
- [ ] Repositório no GitHub com a atividade

## Critérios técnicos mínimos (checklist)

- [ ] As duas páginas foram construídas em HTML.
- [ ] O CSS foi organizado em udois arquivos externos.
- [ ] A estrutura dos elementos foi planejada de forma coerente com o leiaute observado.
- [ ] Os alinhamentos, espaçamentos, proporções e dimensões foram trabalhados para aproximar o resultado do modelo.
- [ ] As duas páginas mantêm padronização visual e estrutural.
- [ ] Os nomes dos arquivos estão claros e coerentes.
- [ ] O código está limpo, legível e sem trechos desnecessários.
- [ ] O repositório está organizado de modo que facilite a validação da atividade.

## Rubricas

### (CT7)

- Insuficiente: Não conseguiu reproduzir as páginas propostas; apresentou estrutura muito diferente do modelo; houve uso incorreto ou muito limitado de HTML e CSS; o resultado ficou desorganizado e distante da composição visual esperada.
- Básico: Reproduziu parcialmente as páginas; demonstrou compreensão inicial da estrutura em HTML e da estilização em CSS, mas com inconsistências visíveis de alinhamento, espaçamento, proporção, hierarquia visual ou fidelidade ao modelo.
- Adequado: Reproduziu corretamente as duas páginas propostas; utilizou HTML e CSS com coerência; os elementos ficaram organizados, com boa aproximação visual em relação ao modelo solicitado.
- Avançado: Além do adequado, alcançou alta fidelidade visual nas duas páginas, manteve consistência entre os arquivos, organizou o CSS com clareza, evitou repetições desnecessárias e demonstrou cuidado técnico e estético na composição final.

### (CS3)

- Insuficiente: Repositório desorganizado; faltam arquivos; a estrutura dificulta a leitura, navegação ou validação da atividade.
- Básico: Organização parcial; os arquivos estão presentes, mas há pouca padronização, pouca clareza na estrutura ou baixa revisão do código.
- Adequado: Estrutura clara, separação correta entre HTML e CSS, arquivos identificados e entrega completa.
- Avançado: Além do adequado, demonstra planejamento técnico, padronização consistente entre as páginas, boa legibilidade do código e cuidado na apresentação final do repositório.

## Como entregar

- Criar um repositório novo no GitHub com o nome `atividade_` (minúsculo, sem acentos, sem espaços, usando `_`).
- Organizar no repositório os dois arquivos HTML e o arquivo CSS da atividade.
- Entregar o link do repositório no ambiente virtual de aprendizagem.
